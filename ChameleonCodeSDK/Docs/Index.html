Augmented Reality Treasure Hunt

This is a high level summary of the Augmented Reality Treasure Hunt which was created Olivia, Andrew, Theo, Davi at the 2016 Build Madison Event.

User Experience

A typical game will involve the user following this general sequence.

  1. Login
    Login for the first time, or log in as returning user

  2. Story
    The narrative that sets up the treasure hunt

  3. Newuser
    Choose a character to play through the hunt

  4. Puzzle
    Contains photo, hint, and "I found it!" button

  5. Treat
    Upon completion of puzzle, user receives treat (increased discount) and earns points for leaderboard

  6. Gameover
    Congratulations! You finished the hunt. You were the nth finisher.

  7. Leaderboard
    Lists usernames and points earned for the player's hunt.

  8. Lifecode
    Enter code from vendor to gain a new life

  9. Redeem
    Redeem discount code at vendor. The vendor can see the coupon - and validate it with a mobile device - if they choose.

System Views

While not in the game, a user might also visit these areas.

  1. App
    Abstract App Root

  2. Home
    User home page - lists possible hunts?

  3. Settings
    User preferences and settings

  4. Logout
    Logout

  5. Admin
    Abstract Admin Root

  6. AdminHome
    Administrative home page

  7. Users
    A list of users planing the game (and all their associated meta data)

  8. User
    A hunt taking place in a city

  9. Hunts
    A list of hunts, taking place in some city

  10. Hunt
    A hunt taking place in a city

  11. HuntPlanner
    A hunt planner which allows one to easily add/edit puzzles for a hunt

  12. Cities
    A list of cities where hunts are

  13. City
    A city in which a hunt may (or may not) take place.

  14. Characters
    A list of charcter templates used

  15. Character
    A character available in a hunt

  16. Locations
    A list of locations a hunt might happen in

  17. Location
    A location within a city.
